# Hapi Player 开发 Prompt

请使用 Rust 开发一个轻量级的“歌词展示跟唱软件”，最终交付一个无需用户编译、可直接双击运行的 Windows `.exe` 文件。软件需要具备友好的 GUI 界面，支持歌词、音频和波形的同步播放与校准。

## 一、交付要求

- 使用 Rust 开发桌面端应用。
- 最终产物必须包含一个可直接运行的 `.exe` 文件，用户无需安装 Rust 或自行编译。
- GUI 需要简洁、直观、用户友好，适合非技术用户使用。
- 程序运行时不应弹出额外的控制台窗口。
- 应尽量避免使用 `unsafe`。如确需使用，必须说明原因；优先使用安全 Rust 实现。
- 需要关注资源释放和内存使用，避免明显的内存泄漏。

## 二、歌词导入与显示

- 支持识别并导入 LRC 格式歌词文件。
- 歌词需要根据时间戳同步滚动播放。
- 当前播放歌词需要居中高亮显示。
- 当前歌词前后各显示 3 条歌词。
- 支持导入任意文本文件形式的歌词，不局限于 `.lrc`。
- 支持普通文本歌词文件，例如 `.txt`、`.text`、`.lyric`、`.lyrics`。
- 当歌词文件没有 LRC 时间标签时，应按行展示歌词。
- 支持多种常见文本编码，避免中文乱码。
- 至少支持以下编码：
  - UTF-8
  - UTF-8 BOM
  - UTF-16LE
  - UTF-16BE
  - GBK
  - GB18030
  - Big5
  - Shift-JIS
  - EUC-KR
  - Windows-1252
- 界面中的中文必须正常显示，不能出现乱码。
- 应主动加载可显示中文的系统字体，例如微软雅黑、黑体、宋体等。

## 三、音频导入与播放

- 支持导入两轨音频文件。
- 尽量不限制音频格式，至少覆盖常见音频格式，例如：
  - MP3
  - WAV
  - FLAC
  - AAC / M4A
  - OGG / Vorbis
- 两轨音频应能同步播放。
- 播放按钮按下后，需要同步播放音频、歌词和波形。
- 支持播放、暂停、停止。
- 支持拖动整体进度条改变当前播放位置。

## 四、波形显示

- 导入音频后，需要显示音频波形。
- 两轨音频应分别显示波形。
- 波形区域需要有一个明显的指针，用于显示当前播放位置。
- 播放时指针应随音频播放实时移动。
- 拖动或跳转播放位置后，波形指针需要同步更新。

## 五、拖拽导入

- 软件需要支持拖拽导入文件。
- 用户可以将歌词文件直接拖入窗口。
- 用户可以将音频文件直接拖入窗口。
- 拖入多个文件时，应自动识别歌词文件和音频文件。
- 拖拽时界面需要有清晰的提示，例如“松开即可导入文件 / Release to import files”。
- 界面中也需要保留手动选择文件的按钮或卡片。

## 六、中英双语界面

- 软件界面需要提供中文和英文两种语言。
- 用户可以在界面上切换语言。
- 切换语言后，按钮、提示、错误信息、空状态文案等界面文字应即时更新。
- 中文界面需要避免乱码。
- 英文界面需要表达自然清晰。

## 七、歌词拖动与时间轴联动

- 歌词文本显示区域需要支持鼠标上下拖动。
- 在歌词区域添加一个上下拖动进度条，方便快速浏览歌词位置。
- 用户拖动歌词文本时，不应立即影响正在播放的音频进度。
- 用户拖动歌词文本期间，只预览歌词位置。
- 当用户松开鼠标后，再将音频播放位置跳转到当前选中的歌词时间戳。
- 右侧歌词进度条拖动时也遵循同样逻辑：
  - 拖动时只预览歌词位置。
  - 松开后再改变音频时间轴。
- 双击某一句歌词时，应立即从该句歌词的时间戳处开始播放音频。

## 八、歌词校准与导出

- 添加“歌词校准”按钮。
- 添加一个用于输入校准秒数的输入框。
- 输入框默认预填 `0`。
- 输入框只允许输入正负整数数字。
- 输入框旁边需要显示秒的单位 `S`。
- 点击“歌词校准”按钮时：
  - 读取输入框中的秒数。
  - 将所有歌词时间戳整体增加或减少对应秒数。
  - 校准后的歌词时间戳不能小于 `0` 秒。
- 在菜单栏或顶部操作区提供“歌词另存为 / Save Lyrics As”功能。
- 导出的文件应为新的 `.lrc` 文件。
- 导出时应使用当前校准后的歌词时间轴。
- 导出格式建议为标准 LRC 格式：

```text
[mm:ss.xx]歌词文本
```

## 九、用户体验要求

- 界面应尽量清晰、现代、易用。
- 主要操作需要一眼可见：
  - 导入歌词
  - 导入音频轨 1
  - 导入音频轨 2
  - 播放 / 暂停
  - 停止
  - 语言切换
  - 歌词校准
  - 歌词另存为
- 当前歌词需要有明显高亮效果。
- 歌词拖动、双击跳播、波形点击跳转等操作需要反馈自然。
- 出错时应显示清晰错误提示。

## 十、质量检查

开发完成后需要进行以下检查：

- 确认项目能够成功编译。
- 确认 release 版 `.exe` 能够生成。
- 确认源码没有使用 `unsafe`，或说明无法避免的原因。
- 检查是否存在明显内存泄漏或资源无法释放的问题。
- 使用静态检查工具检查代码质量。
- 建议运行：

```powershell
cargo check
cargo test
cargo clippy --all-targets -- -D warnings
cargo build --release
```

## 十一、最终产物

最终应提供：

- 可直接运行的 Windows `.exe` 文件。
- 源码项目。
- 简短使用说明。
- 支持中英双语、拖拽导入、双轨音频、波形显示、歌词同步、歌词拖动跳转、歌词校准和 LRC 导出。
